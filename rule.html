Rule naming

Rule ID

The ID of a rule depends on what type of rule it is. The Rule Id is made up of the following parts.

The different parts are separated with dots.

Part Description
MPID This is the management pack ID. This must be included to make the rule ID unique in the management group. To learn more about MPID, see Management pack naming
Description This is a description of the rule.
Rule Type This is the type of rule. Possible values are Alert or Collection.
Suffix This should be Rule.


Using the table above we get the following examples:

Rule Type Rule Information Rule ID (MPID left out)
Collection Rule Collects temperature values from a temperature sensor. MPID.Temperature.Collection.Rule
Alert Alerts if the following events are found:
Log: Application
EventID: 155, 156, 167 		MPID.SyncFailureEvents.Alert.Rule
Eventlog Collect the following event:
Log: Application
EventID: 1432 		MPID.ApplicationEventID1432.Collection.Rule


Display name

The rule display name is harder to set any fixed rules for. The display name is what is shown in the Operations Console.

Based on the examples above the following display names coud be used.

Rule ID (MPID left out) Display Name
MPID.SyncFailureEvents.Alert.Rule Synchronisation failure
MPID.Temperature.Collection.Rule Collect temperature value
